Ruby Hashes vs. JavaScript Objects

Oh Javascript. How you tantalize me with your objects that look just like Ruby except slightly different. Every day you push me closer to the grave.

Here's the difference:

A Ruby hash...

looks like this: 

			
				people_and_my_impressions_of_them = {
				Ryan: "Totally lame",
				Jan: "Super tough",
				Alan: "Get this guy some deodorant",
				Rick: "Great shorts but he wouldn't tell me where he got them"
				}

This is how you access one: 

			
				people_and_my_impressions_of_them[:Jan]
				#=> Super tough

And finally, here's how you add to one: 

			
				people_and_my_impressions_of_them[:Kyle] = "What a guy!"

There are, of course, many different ways to do earch of those things. But I'll leave you with that for now so we can move onto JavaScript!

Here's what a JavaScript object looks like: 

			
				var people_and_my_impressions_of_them = {
				ryan: "Totally lame";
				jan: "Super tough";
				alan: "Get this guy some deodorant";
				rick: "Great shorts but he wouldn't tell me where he got them";
				}

Pretty similar, right? WRONG. LEAVE.

Sorry, I just... JavaScript can be frustrating, guys. Where was I? Ah yes, how to access them: 

			
				console.log(people_and_my_impressions_of_them.jan)

				or

				console.log(people_and_my_impressions_of_them[jan])

				#=> Super tough

Adding to one or changing one: 

			
				people_and_my_impressions_of_them.mike = "What a snooze!"

And deleting from one: 

			
				delete people_and_my_impressions_of_them.mike

Now this is all great, but the difference isn't all syntax. With Ruby hashes, you're creating a list of keys and values. You're doing the same with JavaScript, but in a way, a JavaScript object lends itself better to categories with values. This is because of the easy, custom functions that you define, like ".mike" above. As a result, you might use multiple JavaScript objects with descriptive values. Here's an example of what that might look like, and how you might manipulate them: 

			
				var ian = {
				name: "Ian";
				age: 22;
				opinion: "THE BEST"
				};

				var albert = {
				name: "Albert";
				age: "101";
				opinion: "What an old fart"
				};

				var jan = {
				name: "Jan"
				age: "52"
				opinion: "Seriously, she's the best."
				};


				ian.mood "Sad";
				albert.opinion = "Rest in Peace";
				jan.opinion = "alleged murderer";



				#=> 
				var ian = {
				name: "Ian";
				age: 22;
				opinion: "THE BEST"
				mood: "Sad"
				};

				var albert = {
				name: "Albert";
				age: "101";
				opinion: "Rest in Peace"
				};

				var jan = {
				name: "Jan"
				age: "52"
				opinion: "alleged murderer"
				};

In my opinion, the biggest difference between Ruby hashes and Javascript objects is this: Hashes are easy to make, whereas objects are easy to manipulate.
