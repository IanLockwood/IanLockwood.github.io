Classes in Session

So, if you're reading this blog, I assume you're someone with Ruby experience, but just a little - or you're pretending to be - so you probably know that Ruby is an object oriented language. Strings, arrays, hashes - they're all objects. But what you may not know is that those objects are all instances of classes: the String class, the Array class, and of course, the Hash class.

Note: Integers, floats, and a LOT of other things are objects/instances of classes, but let's stick to our three examples for conciseness.

We've actually worked with classes a little bit already. Whenever we've created a string, array, or hash with the .new method, we've been calling that method on the class. You can tell by the capitalized first letter: 


		String.new = ""
		Array.new = []
		Hash.new = {}

Well that's great and all, but what am I getting at here? Why won't I just get to the point?

Fine, jeez.

You can make your own classes.

Saw whaaaaaaaaaaaaaa?

That's right - you can define your own classes, objects (which, remember, are instances of classes), and even methods to be run exclusively on an object of that class. Let's get to the nitty-gritty. How do you do it, and why? 


			class Name

			end

			new_name = Name.new

Congratulations! You made a class called "Name"! You even (almost) made an object that is an instance of that class!

...

Well, that wasn't very interesting. Of course, the details of and uses for custom classes are countless, but allow me to walk you through some of the basic and interesting ones. 


			class Name

				def initialize(name)

				end

			end

			new_name = Name.new("Burt")

Neat! We wrote a class specific method! Not that specific, mind you - initialize is a common, private method used to initialize the class with an argument. In this case, initialize grabs "Burt" for us, but doesn't do anything with it. Let's fix that. 


			class Name

				def initialize(name)
					@name = name
				end

				def print
					p @name
				end

			end

			new_name = Name.new("Burt")

			new_name.print

			#=> "Burt"

What's going on here? Well, we created an actual custom method, print, which is printing "Burt" for us. That's pretty cool. But I assume what you're huffing and puffing about is @name. Is that a variable? What's with the @ symbol? And if there's anything I know about local variables, it's that they CAN'T pass data between classes!!!

Woah, calm down! I didn't mean to get you all worked up. I'm sorry. But you're on to something - @name is not a local variable. It's an instance variable. Instance variables start with an @ symbol, and unlike local variables, which are scoped to a method, instance variables are scoped to a class, so you can share variables between connected methods in your class. Far out!

I won't unload too much on you today, but to finish up, let's have our class actually do something other than print. 


			class Name

				def initialize(name)
					@name = name
				end

				def sh
					@name = "SH" + @name
				end

				def print
					p @name
				end

			end

			new_name = Name.new("Burt")

			new_name.sh
			new_name.print

			#=> "SHBurt"

Awesome! You made Burt's name sound funny. More importantly, we assigned an instance variable in one method, reassigned it in another, and printed it in yet another. Pretty cool, huh?

There's a lot more to learn about classes, but I think you can see from even this tiny, simple example that the possibilities are endless. Let your imagination run wild!
